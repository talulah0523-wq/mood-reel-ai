# MOOD REEL

> 让此刻，遇见一部电影。

MOOD REEL 是一个根据用户此刻心情推荐电影的产品原型。它不从“想看喜剧还是剧情片”开始，而是把自然语言描述转换成可解释的观影需求，再从本地电影库中选出三部更适合当下的电影。

[在线体验 MOOD REEL](https://mood-reel-talulah.talulah0523.chatgpt.site)

## 为什么做

传统电影推荐更擅长回答“喜欢这部电影的人还喜欢什么”，却很难回答“今晚的我适合看什么”。用户通常需要在片单、简介、评分、短评和相似推荐之间反复跳转，才能判断一部电影是否符合当下的情绪和精力。

MOOD REEL 希望缩短这个过程：

1. 用户描述现在的心情，也可以选择快捷心情。
2. 系统将描述整理为刺激度、节奏、情绪负担、冲突强度和观看门槛等观影线索。
3. 推荐引擎先执行硬过滤，再进行匹配评分与多样性重排。
4. 最终返回三部差异化电影，并解释它们为什么适合此刻。

## 当前功能

- 自然语言心情输入与快捷心情标签
- 初始空状态与输入校验，不使用隐藏的默认推荐
- 200 部本地电影库，无需 API Key 即可体验
- 200 张本地电影海报，访问页面时不调用外部图片 API
- 参考电影搜索、确认和偏好维度选择
- “当前心情优先”与“更接近参考电影”两种推荐策略
- 情绪负担、冲突强度等硬过滤，避免把“安静”误判为“轻松”
- 三部结果的多样性控制
- “换一批”和“换一部”的去重处理
- 基于电影特征的差异化推荐理由与看前提醒
- 选定电影、复制片名和搜索观看渠道
- 键盘操作、ARIA 状态反馈和移动端单列布局

## 推荐逻辑

```text
理解当前心情
→ 生成结构化观影线索
→ 执行硬过滤
→ 召回候选电影
→ 匹配评分
→ 多样性重排
→ 选出三部
→ 生成可解释推荐理由
```

当前心情是推荐的主要依据；参考电影只作为软偏好。即使用户选择了高强度电影作为参考，明确提出的“不要太沉重”等条件仍会优先执行。

## 本地运行

需要 Node.js 22.13 或更高版本，以及 pnpm。

```bash
pnpm install
pnpm dev
```

打开 [http://localhost:3000](http://localhost:3000)。

海报文件保存在 `public/posters`，标题映射由 `app/posters.ts` 提供。需要随片库更新海报时运行：

```bash
pnpm posters:refresh
```

该命令只在维护片库时获取并保存静态图片；用户访问页面时不会调用外部海报接口。

## 验证

```bash
pnpm typecheck
pnpm test
```

测试覆盖基础页面渲染、200 部片库数量、推荐回归场景和生产构建。

## 当前边界

- 当前使用 200 部本地电影数据，适合展示产品与推荐逻辑，不代表完整商业片库。
- 电影海报来自 TMDB 的公开电影资料，仅用于个人学习与作品集展示；不提供播放资源。
- 推荐依据来自结构化规则与本地字段，不会让大模型凭记忆虚构电影事实。
- 项目目前面向个人学习与作品集展示。
